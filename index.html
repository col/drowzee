Put Your Kubernetes Deployments to Sleep

Drowzee is a Kubernetes operator that automatically scales down deployments according to a schedule and wakes them up when needed.

Get Started View on GitHub

Why Choose Drowzee?

Scheduled Scaling

Automatically scale down deployments during off-hours and scale them back up when needed, saving valuable cluster resources.

Web Interface

Manage sleep schedules, view their status, and manually override schedules through an intuitive web interface.

Ingress Redirection

Redirect traffic from sleeping deployments to Drowzee, allowing users to wake up services on demand.

Flexible Deployment

Run in single namespace mode or cluster mode to manage deployments across multiple namespaces.

Installation

Install Drowzee using Helm:

helm repo add drowzee https://col.github.io/drowzee
helm repo update
helm upgrade --install drowzee drowzee/drowzee --namespace default -f values.yaml

To uninstall:

helm uninstall drowzee

Configuration

In single namespace mode, Drowzee will only detect sleep schedules and manage deployments in the same namespace as the Drowzee deployment.

mode: single_namespace

app:
  host: "drowzee.dev.example.com"

secrets:
  secretKeyBase: "random-secret-key"

# Optional but required in order to access the Drowzee Web UI
ingress:
  enabled: true
  annotations:
    kubernetes.io/ingress.class: "nginx"
  hosts:
    - host: drowzee.dev.example.com
      paths:
        - path: /
          pathType: ImplementationSpecific

In cluster mode, Drowzee will detect sleep schedules and manage deployments in multiple namespaces.

mode: cluster

app:
  host: "drowzee.nonprod.example.com"
  # Use `__ALL__` to manage all namespaces
  namespaces: "dev,qa,staging"

secrets:
  secretKeyBase: "random-secret-key"

# Optional but required in order to access the Drowzee Web UI
ingress:
  enabled: true
  annotations:
    kubernetes.io/ingress.class: "nginx"
  hosts:
    - host: drowzee.nonprod.example.com
      paths:
        - path: /
          pathType: ImplementationSpecific

Drowzee is reasonably lightweight but resource requirements depend on the number of sleep schedules and deployments being managed. Here's a safe starting point:

resources:
  requests:
    cpu: 100m
    memory: 256Mi
  limits:
    cpu: 200m
    memory: 256Mi

About Drowzee

Drowzee was inspired by Snorlax, with several improvements:

  • Simpler Ingress Management: Uses a single annotation to redirect traffic rather than rewriting the whole ingress record.
  • Explicit Wake Up: Requires users to click the wake-up button, avoiding unexpected wakeups.
  • Feature-rich UI: Provides a comprehensive web interface to view and manage sleep schedules.

Built with the Bonny framework for creating Kubernetes operators in Elixir.
